# VIP 고객 클러스터링 분석 보고서

## 1. 연구 목표 및 한계
### 연구목표
이 프로젝트는 외부 API 기반의 거래 건 단위 데이터를 활용하여, 고객 식별자 없이도 거래 행태 기반으로 유사 고객을 그룹화하는 것을 목표로 합니다. 이를 통해 거래의 특징적인 패턴을 파악하고, VIP 고객군을 클러스터링하여, 향후 마케팅 전략 및 맞춤형 서비스를 데이터 기반으로 제시할 수 있습니다. 분석에 사용된 KMeans 클러스터링 알고리즘은 거래 금액, 평균 코인가격, 7일 변동성 등을 변수로 사용하여, 유사한 거래 패턴을 가진 고객들을 그룹화합니다.

### 연구의 한계
본 프로젝트에서 빗섬 API를 사용하려 했으나, 몇 가지 한계로 인해 데이터 수집에 어려움이 있었습니다. 빗섬 API는 실시간 거래 내역을 제공하지만, 과거 거래 데이터 조회가 불가능하거나 매우 제한적이었습니다. 또한, API 요청 횟수 제한으로 인해 대규모 데이터 수집이 어려웠습니다. 이로 인해 3개월 이상의 거래 데이터를 확보하는 데 어려움이 있었으며, 이로 인해 VIP 고객군 분석에 필요한 충분한 데이터를 확보하지 못했습니다.

따라서, 빗섬 데이터 대신 Dummy Data를 사용하여 분석을 진행했으나, 실제 빗섬 데이터를 활용할 수 있었다면 더 정확하고 의미 있는 분석이 가능했을 것입니다. 실제 데이터는 고객군 분포와 거래 패턴을 더 잘 반영하고, 실제 고객 행동을 바탕으로 한 분석이 이루어질 수 있을 것입니다.

## 2. 데이터 저장 전략

| 필터링 항목 | 설정값 | 이유 |
|------------|--------|----|
| 거래소 | Upbit, Bithumb | 한국 VIP 고객 중심 연구 |
| 데이터 기간 | 최근 3개월 | VIP 고객의 패턴을 분석하기 충분한 기간 |
| 거래 필터링 | 3개월 동안 거래금액 상위 10%만 저장 | VIP 트레이딩 패턴 분석 최적화 |
| 거래 단위 | 개별 거래 데이터 저장 (합산 X) | 거래 패턴 클러스터링에 필요 |

## 3. 저장 데이터 항목 (Upbit & Bithumb API에서 가져올 항목들)
- **거래된 시간 (timestamp)** → VIP 고객의 거래 집중 시간대 분석
- **거래 가격 (price)** → VIP 고객이 주로 거래하는 가격대 분석
- **거래량 (volume)** → VIP 고객의 대량 거래 패턴 분석
- **매수/매도 방향 (ask/bid)** → VIP 고객의 주 매매 전략 파악
- **거래된 마켓 (market)** → VIP 고객이 주로 거래하는 종목 분석 (BTC, ETH, XRP 등)

## 4. 클러스터링 변수 설정
- **trade_amount (거래 금액)** → VIP 고객의 거래 규모를 구분하기 위해 사용
- **avg_coin_price (평균 코인 가격)** → 거래된 자산의 가격 특성을 반영하기 위해 사용

### 📌 클러스터별 맞춤형 액션 플랜

#### ✅ 일반 VIP 트레이더 (0번) – 거래 빈도 유지를 위한 리워드 제공
- **🚀 실행 전략**:
  - 거래량 기준 VIP 등급 설정 & **수수료 할인 혜택 제공**
  - 특정 거래량 초과 시 **거래 보상 지급 (예: 거래소 토큰 리워드)**
  - **소셜 트레이딩 기능 도입** → 인기 트레이더의 매매를 따라갈 수 있도록 유도

#### ✅ 고위험 투기성 Whale (1번) – 리스크 높은 코인 트레이딩 최적화
- **🚀 실행 전략**:
  - **변동성 높은 코인 전용 트레이딩 상품 출시** (예: 레버리지 조정 가능)
  - Whale 전용 **고주파 트레이딩 최적화 API 지원**
  - Pump & Dump 감지 시스템 도입 → Whale이 한쪽 방향으로 쏠릴 경우 자동 경고

#### ✅ 안정적인 Whale (2번) – 대형 자산 고객 대상 서비스 강화
- **🚀 실행 전략**:
  - **고액 거래 전용 VIP 컨설팅 서비스 제공** (예: 투자 컨설팅, OTC 매칭 서비스)
  - **대형 주문 전용 유동성 풀 운영** → 대량 매매 시 슬리피지 최소화
  - **법인 및 기관 투자자 대상 특별 요금제 도입**

#### ✅ Market Maker (3번) – 거래량 기반 인센티브 제공
- **🚀 실행 전략**:
  - **거래소 유동성 공급 인센티브 프로그램** 운영
  - 특정 마켓에서 일정량 이상의 거래량 유지 시 **수수료 환급 프로그램 도입**
  - **HFT(고빈도 트레이더) 대상 거래 속도 최적화 서비스 제공**

#### ✅ 초대형 개별 거래 (4번) – 기관 투자자 맞춤 서비스
- **🚀 실행 전략**:
  - **프라이빗 트레이딩 데스크 운영** → 대형 투자자 전담 지원
  - **대량 주문 전용 블록 트레이딩 서비스** 제공
  - **OTC(장외거래) 플랫폼 연계**하여 대형 주문에 따른 시장 영향 최소화

## **8. 향후 연구 개선 방향 및 추가 분석 아이디어**
VIP 고객의 거래 패턴을 더욱 정교하게 분석하고 맞춤형 혜택 전략을 최적화하기 위해, 향후 다음과 같은 추가 연구가 유의미할 수 있음.

### 8-1. 향후 연구 개선 방향

#### 1) 변수 고도화
#### 변수 고도화 제안 표

| **변수명**               | **설명**                                                                                   | **현재 사용 여부** | **향후 추가 제안**                                                 |
|--------------------------|--------------------------------------------------------------------------------------------|---------------------|---------------------------------------------------------------------|
| **trade_amount**          | 거래 금액 (가격 × 거래량)                                                                   | 사용됨               | 추가 불필요, 핵심 변수                                             |
| **avg_coin_price**        | 거래 금액 대비 평균 코인 가격                                                               | 사용됨               | 추가 불필요, 핵심 변수                                             |
| **trade_frequency**       | 거래 빈도 (특정 기간 내 거래 횟수)                                                           | 미사용               | 고객의 거래 빈도 파악, 고빈도 거래자 구분 가능                     |
| **price_7d_volatility**   | 최근 7일간의 **가격** 변동성                                                                 | 미사용               | 가격의 변동성 분석, 변동성이 큰 고객군을 구분 가능                  |
| **market_condition**      | 시장 상태 (상승/하락 등 시장의 변화에 대한 반응)                                             | 미사용               | 시장의 전반적인 상황에 대한 반응을 고객군 분석에 반영              |
| **trade_time_of_day**     | 거래 시간대 (시간대별 거래량)                                                               | 미사용               | 거래 시간대에 따른 패턴을 분석하여 시간대별로 고객군을 세분화 가능   |
| **price_change_pct**      | 코인 가격의 변화율 (과거 대비 가격 변화 비율)                                               | 미사용               | 가격 변화에 민감한 고객을 분류 가능                                |
| **avg_trade_volume**      | 평균 거래량 (과거 특정 기간 동안의 거래량 평균)                                              | 미사용               | 거래량 패턴을 통해 고빈도 거래자, 저빈도 거래자 구분 가능           |
| **trade_category**        | 거래의 성격 (매수/매도 구분)                                                                | 미사용               | 매수와 매도 패턴을 구분하여 분석, 매도 경향이 높은 고객군 추출 가능  |

### 변수 설명:

- **trade_frequency**: 특정 기간 내 거래 횟수를 계산하여 **고빈도 거래자**를 구분할 수 있습니다. 이를 통해 **트렌드 반응**이 빠른 고객군을 파악할 수 있습니다.
- **price_7d_volatility**: 최근 **7일간 가격**의 변동성을 측정하여, **가격 변동성에 민감한 고객군**을 식별할 수 있습니다. 변동성이 큰 고객은 **단기적인 시장 반응**을 나타낼 수 있습니다.
- **market_condition**: 시장이 상승 또는 하락할 때의 고객 반응을 분석하여 **시장의 변화에 민감한 고객군**을 파악할 수 있습니다.
- **trade_time_of_day**: 거래가 일어나는 **시간대별 패턴**을 파악하여 특정 시간대에 활동이 많은 고객을 추적할 수 있습니다.
- **price_change_pct**: **가격 변화율**을 사용하여 가격 변화에 민감하게 반응하는 고객을 분류할 수 있습니다.
- **avg_trade_volume**: 고객의 **거래량** 패턴을 파악하여 **고빈도 및 저빈도 거래자**를 구분할 수 있습니다.
- **trade_category**: **매수/매도** 성격에 따라 고객을 구분하여, **매도 비율**이 높은 고객군을 파악할 수 있습니다.

#### 2) Elbow Method 활용 최적 K 값  
본 연구에서는 KMeans 클러스터링을 사용하여 고객군을 분류했지만, Elbow Method를 통해 최적의 K 값을 도출하지 못한 점이 아쉬웠습니다. Elbow Method를 활용하여 WCSS (Within-Cluster Sum of Squares) 값을 기반으로 **클러스터 수 (k)**에 대한 최적값을 도출하면, 보다 정확한 클러스터링을 할 수 있습니다. Elbow 부분을 그래픽으로 시각화하여, 데이터에 가장 적합한 클러스터 수를 선정하고, 이를 통해 고객군을 더 세분화하며 분석의 신뢰도를 높일 수 있습니다.

### 8-2. 추가 분석 아이디어

#### **1️⃣ VIP 고객별 수익성 분석 (Profitability & PnL)**
- 현재 데이터로는 개별 고객의 실현 수익률(ROI)을 정확히 계산하기 어렵지만, 향후 **잔고 데이터 및 입출금 내역**이 확보된다면 더욱 정밀한 수익성 분석이 가능.
- 이를 통해 **손실을 자주 보는 고객군을 위한 맞춤형 리스크 관리 지원**, 혹은 **거래량 대비 수익률이 낮은 고객을 위한 수수료 인센티브 설계**가 가능.

#### **2️⃣ 거래소 간 고객 이동 패턴 심화 분석**
- VIP 고객이 Upbit과 Bithumb 사이에서 거래소를 이동하는 원인을 보다 정밀하게 분석하면 **거래소 간 경쟁 전략 최적화가 가능**.
- 특히 **거래소별 수수료 차이, 특정 코인의 유동성 및 신규 상장 여부가 이동에 미치는 영향을 측정**하면, VIP 고객 유지 및 유치 전략을 수립하는 데 활용 가능.

#### **3️⃣ BTC 가격 변동과 VIP 고객의 반응 분석**
- 현재 데이터로는 VIP 고객이 BTC 가격 변동에 어떤 패턴으로 반응하는지를 직접 분석하기 어렵지만, 향후 **BTC 변동성 데이터와 특정 클러스터의 거래 행태를 결합**하면 더욱 강력한 인사이트 도출이 가능.
- 이를 통해 **BTC 급락 시에도 거래를 지속하는 VIP 고객을 대상으로 한 리스크 보호 혜택 제공**이나, **Whale의 매도·매수 패턴을 기반으로 한 시장 예측 모델 구축**이 가능할 것으로 예상됨.

이러한 연구 방향을 추가적으로 고려하면 VIP 고객 분석의 정밀도를 더욱 높이고, 보다 전략적인 혜택 설계가 가능할 것임.
